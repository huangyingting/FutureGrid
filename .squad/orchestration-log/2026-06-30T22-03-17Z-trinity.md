5. Mobile fallback: stacked narrative with inline charts (each chart mounts once in visible container)

## Post-Batch: Production Incident Discovery

During real-browser smoke test on completed Batch 7, error boundary caught crash in `/report`:
- **Error:** "missing: 0" from d3-sankey in SkillFlowSankey
- **Root:** Pre-existing bug (Batch 6): link ids built as numeric indices, but d3-sankey configured with nodeId(d => d.id) expecting strings
- **Exposure:** ReportView mounted charts twice (sticky + display:none lg:hidden), so crash occurred on mobile in zero-width container

**Fix Applied:** d915eea hotfix
1. SkillFlowSankey: links now use string ids
2. ReportView: scrollytelling only on lg+; mobile renders stacked narrative with inline charts (each chart mounts once in visible container)
3. Added SkillFlowSankey render regression test

**Lesson:** jsdom tests don't catch D3 render crashes (ResizeObserver mocked). Always do headless Chrome smoke test before ship.

✅ HOTFIX VERIFIED — Headless Chrome desktop + mobile viewports, no error boundary
