## Scope

Build client-side CSV + JSON data export widget for `/sources` page. Three datasets: Occupations, Countries, Sources. No new dependencies (inline CSV helper).

## Approach

1. Created `components/sources/DataExport.tsx` (client, "use client")
2. Three export buttons (Occupations, Countries, Sources) × 2 formats (CSV, JSON)
3. Data sources:
   - **Occupations:** `generateAllCareerInsights()` → 10 flat CSV fields (excludes skills[], employmentHistory, wageHistory arrays)
   - **Countries:** `getCountryMapData()` → 10 flat CSV fields (excludes diffusionTrend, readinessSubIndices)
   - **Sources:** `getDataSources()` → 6 flat CSV fields from `.sources[]`
4. CSV implementation: Inline `toCsv()` + `csvEscape()` helpers (no dependency on CSV lib)
5. Filenames: `futuregrid-{key}-2025.{csv|json}` (e.g., `futuregrid-occupations-2025.csv`)
6. File download: `URL.createObjectURL()` + programmatic click (SSR-safe, no document access at render/module level)
7. i18n: 12 keys (downloadLabel, csvFormat, jsonFormat, occupationsDataset, countriesDataset, sourcesDataset, status labels)
8. Accessibility: `aria-live="polite"` announcement region, `aria-label` on buttons, `aria-busy` on loading

### Decisions

1. **No CSV library:** Inline helpers only — `toCsv()` builds header row + data rows, `csvEscape()` quotes fields with comma/quote/newline, doubles inner quotes
2. **SSR safety:** All `document`, `URL.createObjectURL`, `Blob` calls isolated to `handleDownload()` click handler (never at module/render top-level)
3. **Styling:** `.glass` panel + button classes copied from SourcesView.tsx / ExploreView.tsx (consistent aesthetic)
4. **Accessibility:** `aria-live` region announces download start; buttons labeled with dataset + format; loading state tracked via `aria-busy`
5. **Integrator requirement:** Neo registers namespace + places DataExport in SourcesView
