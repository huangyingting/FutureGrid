**Fix Deployed:** d915eea  
**Timestamp:** 2026-06-30T22-03-17Z

## Incident Details

### Error Signature

```
Error: missing: 0
  at d3-sankey internals (link validation)
  → caught by React error boundary
```

### Root Cause Analysis

**Pre-existing bug from Batch 6 (#26):**

1. `SkillFlowSankey.tsx` calls `d3-sankey` with `.nodeId(d => d.id)` (string ID extractor)
2. But links are built using NUMERIC INDICES (0, 1, 2, ...) instead of string IDs
3. d3-sankey validates each link target against nodeId function → expects string IDs → throws "missing: 0"
4. **Why tests didn't catch it:** jsdom mocks `ResizeObserver` → d3 layout code (runs inside ResizeObserver callback) never executes in vitest → error never surfaces

### Secondary Bug Exposed in Batch 7

**ReportView component (Trinity, #28):**
- Mounted each chart twice: sticky panel + `display:none lg:hidden` display state
- On mobile devices (below lg breakpoint): sticky panel + hidden duplicate chart
- Hidden chart mounted in 0-width container (parent set to `display:none`)
- D3 layout path runs anyway → crashes in invisible container
- SkillFlowSankey error surfaced here

## Fixes Applied

### Fix 1: SkillFlowSankey Link IDs (Pre-existing Bug)

**File:** `components/charts/SkillFlowSankey.tsx`

**Change:** Links now built with string IDs instead of numeric indices

```javascript
// Before (BROKEN):
const links = data.skillTransitions.map((trans, idx) => ({
  source: idx,  // numeric index
  target: idx + 1,  // numeric index
  value: trans.count
}));

// After (FIXED):
const links = data.skillTransitions.map(trans => ({
  source: trans.source.id,  // string ID
  target: trans.target.id,  // string ID
  value: trans.count
}));
```

### Fix 2: ReportView Mobile Layout (Batch 7 Exposure)

**File:** `components/report/ReportView.tsx`

**Change:** Scrollytelling only rendered on desktop (lg+); mobile renders stacked narrative with inline charts

```javascript
// Before:

// Mobile: same sticky panel (zero-width) + display:none duplicate chart

// After:
// Desktop (lg+): sticky chart + narrative (charts mount once in visible container)
// Mobile/SSR: stacked narrative blocks with inline charts
// Each chart mounts exactly once in a visible container
```

## Lessons Recorded

### Lesson 1: jsdom Test Gap for D3

**Finding:** 103/103 vitest tests pass, but D3 charts crash in real browser.

**Root Cause:** jsdom mocks ResizeObserver. D3 layout code runs inside ResizeObserver callback → never executed in jsdom environment → errors never surface in test.

**Implication:** Unit tests alone insufficient for D3-heavy work. Real-browser smoke test is mandatory gate.

### Lesson 2: Container Width Matters for D3

**Finding:** D3 charts crash when mounted in 0-width containers (e.g., `display:none` parent).

**Root Cause:** D3 layout (e.g., d3-sankey simulation) tries to read container dimensions; throws when width/height = 0.

**Implication:** Always mount charts in visible containers. Avoid duplicate mounts in hidden display states.

### Lesson 3: Testing Strategy for Charts

**Going Forward:**
1. Unit test: JSX renders without error (component alive)
2. Regression test: D3-specific nodes appear (layout ran)
3. Real-browser smoke test: Render in headless Chrome at multiple viewports (catch ResizeObserver gaps)
